# 勤怠管理システム 詳細仕様書 (Project Details)

## 1. データベース設計について
本システムは PostgreSQL を使用し、以下の主要なテーブルで構成されています。用途ごとに分類しています。

### 1.1 データベース設計の基本方針（論理削除）
本システムでは、誤操作によるデータ喪失防止と監査目的のため、原則として**論理削除**（ソフトデリート）を採用しています。
各テーブルには `is_deleted` または `is_active` フラグが設けられており、画面上から「削除」や「取消」操作を行っても物理的なレコード削除（DELETE句）は実行されず、フラグの更新（UPDATE句）が行われます。

### 1.2 ユーザー・マスタ系
- `users`: ユーザー情報（ログイン情報、権限、有給設定、アカウントロック情報など）
- `holidays`: 祝日マスタ
- `work_schedule_classes`: 勤務クラス（所属するシフトごとの所定労働時間、休憩時間マスタ）
- `system_settings`: システム全体の共通設定値（キー・バリュー形式）

### 1.3 勤怠・申請系
- `attendance_records`: 日次の出退勤記録（開始・終了時刻、休憩時間、残業時間、深夜労働時間など）
- `attendance_submissions`: 月次勤怠の申請・承認ステータス（対象年月、ステータス、アクション履歴）
- `attendance_correction_requests`: 承認済み勤怠に対する修正申請（差戻し・変更ワークフロー用）
- `leave_applications`: 休暇申請（有給休暇、欠勤、特別休暇など）
- `overtime_records`: 残業予定および実績の申請
- `paid_leave_balance`: 各ユーザーの有給休暇の年次残高（年度ごとの付与・利用・繰越の管理）

### 1.4 承認フロー・通知・ログ系
- `attendance_department_approvers` / `attendance_user_approvers`: 部門ごともしくは個別のユーザーごとの承認者（上長）マッピング
- `user_notifications`: ユーザー向けに表示されるダッシュボード通知（リマインドなど）
- `admin_announcements`: 管理者から全ユーザーに向けて掲示するお知らせメッセージ
- `audit_logs`: 承認・変更・削除などの重要操作を記録するシステム監査ログ

### 1.5 初期データ・サンプルデータ
- `src/main/resources/db/sample/V2__Sample_Data.sql`: 開発やテスト環境向けに、以下のテスト用データや初期設定を自動投入するスクリプトです。
  - **ユーザーアカウント**: 管理者（`admin@example.com` / パスワード: `admin123`）、一般ユーザー（`user@example.com` / パスワード: `user123`）、夜勤テストユーザー（`test@example.com` / パスワード: `test123`）の3名。
  - **勤務クラス（シフト）マスタ**: 「標準勤務」「早番」「遅番」「時短勤務」「夜勤」の5パターンの所定時間設定。
  - **システム共通設定**: 勤怠の締め日設定（毎月21日〜翌月20日）、CSVファイル名パターン、有給付与タイミングやリマインダー通知実行日など。
  - **各種トランザクションデータ**: 早出・遅出・残業・夜勤を含む様々なパターンの出退勤履歴、休暇申請、残業記録、および有給休暇の初期残高とダッシュボード用お知らせメッセージ。

## 2. 画面毎に実装されている機能について

画面（Thymeleafテンプレート）は、共通レイアウト (`layout/base.html`)、一般ユーザー向け(`user/`)、管理者向け(`admin/`)、その他に分かれています。
スタイルは Tailwind CSS で構成されており、ビルド時に `tailwind.css` が生成されます。
ロール別のアクセス制御は Spring Security および `@PreAuthorize` アノテーションにより実施されています。

- **URL**: `/admin/settings`
- **テンプレート**: `admin/settings.html`
- **遷移経路**: サイドバー「システム設定」リンク
- **入力機能**:
  - **勤怠期間設定**: 締め日設定（開始日・終了日）の数値入力（1〜28）
  - **CSVファイル名パターン**: ファイル名テンプレート文字列（例: `{yyyy}-{MM}_{userId}_{name}`）の入力
  - **アラート・バッチパラメータ**: アラート対象値、リマインド配信日時設定
  - **保存**: それぞれの「保存」ボタン
- **出力機能**:
  - 現在のシステム設定パラメータ値の一覧
