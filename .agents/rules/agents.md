# agents.md

このファイルは、AGENT がこのプロジェクトでコード提案を行う際のルールです。

## 1. プロジェクト前提

- Java 25 + Spring Boot 4.x + Maven3.8.x + PostgreSQL 18.x を前提にする
- 画面は Thymeleaf テンプレートを使用からTailwind CSSへ変更する
- DBアクセスは MyBatis Mapper XML を利用する
- マイグレーションは Flyway (src/main/resources/db/migration) を利用する
- 勤怠・残業・休暇などの業務ロジックは既存 of ドメイン用語に合わせる

## 2. まず守ること

- 日本語での応答をする
- 提案は短く、トークンの消費を抑える
- 既存の命名・パッケージ構成・レイヤ分割を維持する
- 変更は最小限にし、無関係なリファクタリングは行わない
- 既存の Controller -> Service -> Mapper の責務を崩さない
- 既存メソッドやテンプレートの公開仕様を、明示的な要求なしに変えない
- 既存のコードスタイル（スペース、インデント、改行）を維持する

## 4. LLM への期待出力

- 変更理由を短く説明し、影響範囲を明確化する
- パフォーマンスよりもまず正確な勤怠計算を優先する
- 提案時は「何を」「なぜ」「どこまで変更したか」を明確にする
- 複数案がある場合は、推奨案を1つ示した上で代替案を簡潔に示す
- 不明点は推測で断定せず、前提を明記する
- 既存コードのどこを参考にしたかを明記する
- 変更内容が既存コードスタイルや命名規則に沿っているかを確認する
- 変更内容が既存のロジックやフローを壊していないかを確認する
- 変更内容が既存のテストケースを壊していないかを確認する
- 変更内容が既存のドキュメントやコメントを壊していないかを確認する
- 変更内容が既存のセキュリティやパフォーマンスを壊していないかを確認する
- 変更内容が既存の依存関係やライブラリを壊していないかを確認する
- 変更内容が既存のビルドやデプロイプロセスを壊していないかを確認する
- 変更内容が既存のユーザーインターフェースやユーザーエクスペリエンスを壊していないかを確認する
- 変更内容が既存の業務ロジックやドメインモデルを壊していないかを確認する
- 変更内容が既存のコードレビューやコーディング規約を壊していないかを確認する
- 変更内容が既存のプロジェクトのスケジュールやリソースを壊していないかを確認する
- 変更内容が既存のプロジェクトの品質やセキュリティを壊していないかを確認する

## 5. 禁止事項

- 認証・認可ロジックの弱体化
- 監査に必要な記録の削除
- 機密情報のハードコード
- 大規模な依存追加を無断で行うこと
- 既存のコードスタイルや命名規則を無視すること
