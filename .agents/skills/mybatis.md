---
name: mybatis-sql-rules
description: MyBatis Mapper XML および SQL を作成・変更する際のルール。SQL設計の基本原則、SQLインジェクション対策、パフォーマンス、可読性、再利用性、既存スタイルとの整合性など、SQL品質を確保するためのルール。SQL変更時のパフォーマンスと構造影響、SQLの安全とバインド、パフォーマンス、可読性、再利用性、既存スタイルとの整合性などのルールに従ってSQLを作成・変更する。
---

- **Mapperの一致**: Mapper Interface と Mapper XML のメソッド名・引数・戻り値を一致させる。
- **パフォーマンスと構造影響**:
  - SQL変更時は既存インデックス・結合条件・ソート順への影響を確認する。
  - N+1 が発生しない取得方式を優先する。
  - 既存のSQLパフォーマンスチューニングパターン（インデックス利用、結合順序、集約関数）を維持する。
- **SQLの安全とバインド**:
  - 文字列連結でSQLを組み立てず、必ずパラメータバインド（`#{param}` などの既存スタイル）を使う。
  - 既存のSQLセキュリティパターン（SQLインジェクション対策、パラメータバインド、エスケープ処理）を維持する。
  - 悲観的ロックを最小の影響で実装をする。
- **where 条件**: 業務上意味のある where 条件を省略しない。
- **コードスタイルと構造**:
  - 既存のSQL構造（サブクエリ、CTE、ウィンドウ関数）を維持する。
  - 既存のSQLスタイル（大文字・小文字、改行位置、インデント）を維持する。
  - 既存のSQLコメントスタイル（`--`、`/* */`）を維持する。
  - 既存のSQL命名規則（テーブル名、カラム名、エイリアス）を維持する。
- **例外処理とログ**:
  - 既存のSQL例外処理パターン（try-catch、throws宣言）を維持する。
  - 既存のSQLロギングパターン（ログレベル、メッセージ構成）を維持する。
- **勤怠計算・検索遅延**:
  - 一覧画面の検索・集計で遅延が出る場合はSQL改善を提案する。
