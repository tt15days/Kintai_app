## ルール一覧

- **Null 安全の意識**: Null 安全を意識し、Optional 濫用よりも事前バリデーションを優先する。
- **例外処理**: 例外は握りつぶさず、業務エラーとシステムエラーを分ける。既存の例外処理パターン（try-catch、throws宣言）を維持する。
- **ログ出力**:
  - ログメッセージは定数として管理し、必要に応じてパラメータを埋め込む。
  - ログには個人情報やパスワードを出力しない。
  - ログにはWindowsOSの絶対パスを出力しない。
- **共通処理・既存資産の再利用**:
  - 既存のユーティリティクラスや共通処理を優先して再利用する。
  - 既存のドメインモデルやDTOを優先して再利用し、新規クラスは必要に応じて追加する。
  - 既存の定数クラスやEnumを優先して再利用し、新規定数は必要に応じて追加する。
- **Lombokの利用**: Lombokの利用は既存コードに合わせる（既存コードがLombokを使っている場合は新規コードもLombokを使う）。
- **日付時刻**: 日付時刻は `java.time` を利用し、タイムゾーンは `application.yml` 設定と整合を取る。
- **定数化**: マジックナンバーや固定文字列は定数化を検討する。
- **コードスタイル**: 既存のコードスタイル（スペース、インデント、改行）や命名規則（クラス名、メソッド名、変数名）を維持する。
